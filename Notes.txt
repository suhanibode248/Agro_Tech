1. PROJECT DESCRIPTION
================================================================================

KrishiOS (Agri-OS) is a comprehensive, AI-powered agricultural operating system
designed specifically for Indian farmers. It provides an end-to-end digital
solution covering crop management, market access, financial services, and
community support.

Core Purpose:
  • Empower small and marginal farmers with technology
  • Reduce intermediaries between farmers and buyers
  • Provide real-time market intelligence
  • Offer AI-driven agricultural recommendations
  • Enable direct marketplace transactions

Target Users:
  ✓ Small & marginal farmers
  ✓ Agricultural buyers/traders
  ✓ Farm cooperatives
  ✓ Government agricultural officers

Key Innovation:
  • All 40 features in a single lightweight application (~258 KB)
  • Completely offline-capable with Progressive Web App (PWA)
  • AI integration via OpenRouter free models (no expensive API costs)
  • Dark green agricultural theme for farmer-friendly interface
  • Responsive design for feature phones to desktops

Technology Stack:
  • Frontend: Pure HTML5, CSS3, Vanilla JavaScript (ES6+)
  • No frameworks or dependencies (lightweight & fast)
  • Service Workers for offline capability
  • OpenRouter API for AI features (optional)
  • Browser LocalStorage for data persistence

Deployment Model:
  • Standalone web application
  • Can be deployed on any web server
  • Works as Progressive Web App on mobile
  • Zero backend required (optional AI via OpenRouter)

Geographic Focus:
  • India-centric (crops, prices, government schemes)
  • Support for 12 major Indian crops
  • Indian MSP (Minimum Support Price) data
  • Indian government schemes and subsidies
  • Local storage facilities and market data

Data Centric:
  • Real-time market prices from major agricultural markets
  • Historical price trends for 12 crops
  • Weather data integration
  • Soil quality parameters
  • Government scheme information
  • Insurance and credit information

3. HOW TO RUN PROJECT
================================================================================

METHOD 1: VS CODE LIVE SERVER (RECOMMENDED & EASIEST)
──────────────────────────────────────────────────────

Steps:
  1. Download & extract the agri-os folder
  2. Open the folder in VS Code (File → Open Folder)
  3. Install "Live Server" extension by Ritwick Dey (search in Extensions)
  4. Right-click on index.html → "Open with Live Server"
  5. Browser opens automatically at http://localhost:5500
  6. App is now running! Start using all features.

Live Server Auto-Reload:
  • Save any file → page auto-refreshes
  • Perfect for development
  • No need to manually refresh

METHOD 2: DIRECT BROWSER OPEN (EASIEST - NO SETUP)
───────────────────────────────────────────────────

Steps:
  1. Extract the agri-os folder
  2. Find index.html in the folder
  3. Double-click it
  4. App opens in your default browser
  5. Ready to use immediately

Note: Works offline, but no auto-reload on file changes.

METHOD 3: PYTHON HTTP SERVER (QUICK SETUP)
────────────────────────────────────────────

Prerequisites: Python 3 installed

Steps:
  1. Navigate to agri-os folder in terminal/command prompt
  2. Run: python -m http.server 8000
  3. Open browser: http://localhost:8000
  4. Works with Python 2 & 3

METHOD 4: NODE.JS HTTP SERVER
──────────────────────────────

Prerequisites: Node.js installed

Steps:
  1. Navigate to agri-os folder in terminal
  2. Run: npx http-server
  3. Open browser: http://localhost:8080
  4. App is running

METHOD 5: WEB SERVER DEPLOYMENT (PRODUCTION)
──────────────────────────────────────────────

For Apache Server:
  1. Copy agri-os folder to /var/www/html/
  2. Ensure .htaccess or server allows access to all files
  3. Access via: http://yourdomain.com/agri-os/

For Nginx:
  1. Copy agri-os folder to /usr/share/nginx/html/
  2. Update server block to point to folder
  3. Restart nginx: sudo systemctl restart nginx
  4. Access via: http://yourdomain.com/agri-os/

For Cloud (AWS S3, Azure Blob, Google Cloud Storage):
  1. Upload all 6 files to cloud storage
  2. Enable "Make Public" or set proper permissions
  3. Enable CORS if needed
  4. Access via cloud storage URL

METHOD 6: ENABLE AI FEATURES (OPTIONAL)
─────────────────────────────────────────

To use AI features (Quality Grader, Disease Detection, etc.):

  1. Visit https://openrouter.ai/
  2. Sign up (free account)
  3. Go to Dashboard → Copy API key
  4. Open app.js in text editor
  5. Find line 3: const OPENROUTER_API_KEY = 'YOUR_OPENROUTER_API_KEY';
  6. Replace with your actual key: const OPENROUTER_API_KEY = 'sk-or-v1-xxxx';
  7. Save file
  8. Reload app in browser
  9. AI features now active!

Free AI Models Available:

  • mistralai/mistral-7b-instruct:free (faster)
  • google/gemma-2-9b-it:free (lighter)

4. HOW IT WORKS (TECH FLOW)
================================================================================

A. ARCHITECTURAL OVERVIEW
──────────────────────────

Single Page Application (SPA) Architecture:

  Browser                    App.js                     LocalStorage
    │                          │                              │
    ├─ Click Button ──────→ Event Handler ─ Store Data ─→ Save to Browser
    │                          │                              │
    ├─ Page Change ──────→ DOM Update ──── Load Data ──→ Retrieve from Browser
    │                          │                              │
    ├─ Form Submit ──────→ Validation ──── Process ────→ Display Results
    │                          │
    ├─ AI Request ──────────────┼──────→ OpenRouter API
    │                          │
    └─ Display Results←────────────────── API Response

B. DATA FLOW DETAILED
──────────────────────

1. User Registration Flow:
   Farmer fills form → Validation in app.js → Stored in LocalStorage
   Data saved as: { farmer_id, name, phone, email, land_size, crops... }
   Persistent across browser sessions

2. Crop Planning Flow:
   User selects crop → App checks LocalStorage for soil data
   → Calculates yield prediction (or calls AI) → Shows calendar
   → User records activities → Saved to LocalStorage

3. Disease Detection Flow (with AI):
   User describes symptoms → Form submission → app.js → OpenRouter API
   → LLM analyzes symptoms → Returns diagnosis & treatment
   → Display results in UI → Store in LocalStorage

4. Marketplace Flow:
   Farmer lists crop → Stored in LocalStorage
   → Displayed to buyers → Buyer sends offer → Negotiation → Transaction
   → Both parties can rate each other → Review stored

5. Weather Integration Flow:
   static weather data in data.js → Displayed in Weather page
   → User activities updated based on weather advice
   → Recommendations shown in Dashboard

C. COMPONENT INTERACTIONS
──────────────────────────

Navigation Layer:
  index.html (navigation code)
    ↓
  app.js → showPage() function
    ↓
  Changes display property of divs (hide/show)
    ↓
  UI updates instantly

Form Handling:
  HTML Form → addEventListener("submit") → Validation
    ↓ (if valid)
  Process data → Store in LocalStorage → Update UI
    ↓ (if invalid)
  Show error messages

D. STATE MANAGEMENT (NO EXTERNAL LIBRARY)
───────────────────────────────────────────

All state managed in app.js:
  • Global variables: currentUser, currentPage, sessionData
  • LocalStorage: persisting user data across sessions
  • DOM: UI components are state representations

State Update Pattern:
  1. User Action → Triggered event
  2. Event Handler → Process in app.js
  3. Update Global State → Change variables
  4. Update LocalStorage → Persist data
  5. Render UI → Update DOM
  6. Display Results → Show to user

E. API INTEGRATION (OPTIONAL)
───────────────────────────────

When API key configured (app.js line 3):

  1. User triggers AI feature
  2. app.js checks API key exists
  3. Formats request → Creates prompt
  4. Sends to OpenRouter API via fetch()
  5. Waits for response
  6. Parses LLM response
  7. Extracts relevant information
  8. Updates UI with results
  9. Stores in LocalStorage (optional)

Example: Quality Grader
  User uploads image → Convert to base64 (if needed)
  → Send to API with crop data → AI grades quality
  → Returns: Grade (A/B/C) + Market Value + Recommendations
  → Display in modal → Save transaction history

F. STORAGE & PERSISTENCE
──────────────────────────

LocalStorage Keys Used:
  • farmers: Array of farmer objects
  • buyers: Array of buyer objects
  • crops: User's crop planning data
  • marketplaces: Listings data
  • transactions: Transaction history
  • user_preferences: UI preferences
  • price_alerts: User's price alerts

LocalStorage Quota:
  • ~5-10 MB per origin (browser dependent)
  • Currently using < 1 MB (plenty of space)
  • Auto-compacted when data removed

Data Backup Strategy:
  • Users can export LocalStorage as JSON
  • Can manually backup via browser DevTools
  • Lost if browser cache cleared (warn users)

G. OFFLINE FUNCTIONALITY (SERVICE WORKER)
──────────────────────────────────────────

sw.js enables offline mode:

  1. Service Worker registered in index.html
  2. Caches static assets on first load
  3. User goes offline
  4. Service Worker serves cached files
  5. All pages load from cache
  6. Forms still work (data saved locally)
  7. When online → Sync with API (if needed)

Offline Features:
  ✓ All pages accessible
  ✓ Existing data viewable
  ✓ Forms can be filled (saved locally)
  ✓ Calculations work
  ✗ API features not available
  ✗ Fresh data not loaded

H. PAGE RENDERING FLOW
────────────────────────

Example: User clicks "Crop Planning"

  1. Navigation click → window.showPage('cropPlanning')
  2. app.js showPage() function called
  3. Hide all visible pages: .style.display = 'none'
  4. Show selected page: .style.display = 'block'
  5. Populate page data from LocalStorage (if exists)
  6. Attach event listeners to form elements
  7. Display animations/transitions (CSS)
  8. Page fully rendered for user interaction

I. VALIDATION FLOW
────────────────────

Form Submission Example:

  User fills form → Clicks Submit
    ↓
  JavaScript captures event.preventDefault()
    ↓
  Validates required fields (empty check)
    ↓ (if error)
  Display error message in red
    ↓ (if valid)
  Validates data types (number, email, etc.)
    ↓ (if error)
  Display format error
    ↓ (if valid)
  Process data → Save to LocalStorage
    ↓
  Show success message
    ↓
  Clear form or navigate away

6. COMMON ISSUES & FIXES
================================================================================

ISSUE #1: Blank Page / App Not Loading
────────────────────────────────────────

Symptoms:
  • Browser shows blank white page
  • No error in console
  • App never loads

Possible Causes:
  1. JavaScript disabled in browser
  2. Browser too old (IE 11, etc.)
  3. File path incorrect
  4. Corrupted index.html file

Solutions:
  ✓ Check: Browser → Settings → Privacy → Enable JavaScript
  ✓ Try: Different browser (Chrome, Firefox, Safari)
  ✓ Verify: All 6 files in same folder
  ✓ Check: Browser DevTools (F12) → Console for errors
  ✓ Re-download: Fresh copy of agri-os folder

ISSUE #2: Styles Not Loading / Page Looks Broken
──────────────────────────────────────────────────

Symptoms:
  • All text, no formatting
  • Colors missing
  • Layout broken
  • Buttons look like plain text

Possible Causes:
  1. style.css file missing
  2. Incorrect file path in index.html
  3. Browser cache issue
  4. CSS file corrupted

Solutions:
  ✓ Verify: style.css in same folder as index.html
  ✓ Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  ✓ Clear cache: DevTools → Network → Disable cache → Refresh
  ✓ Try: Different browser
  ✓ Check: File permissions (readable)

ISSUE #3: Forms Not Saving Data
────────────────────────────────

Symptoms:
  • Fill form → Submit → Data disappears
  • Page refresh loses all entries
  • No error shown

Possible Causes:
  1. LocalStorage disabled
  2. Incognito/Private mode (no storage)
  3. Browser LocalStorage quota exceeded
  4. Browser storage not supported

Solutions:
  ✓ Use Normal mode (not Incognito/Private)
  ✓ Check: Browser settings → Privacy → Storage
  ✓ Check: Console → localStorage available?
    Test: localStorage.setItem('test', 'value'); localStorage.getItem('test');
  ✓ Clear old data: DevTools → Application → LocalStorage → Delete
  ✓ Try: Different browser

ISSUE #4: AI Features Not Working
──────────────────────────────────

Symptoms:
  • Click AI button → Nothing happens
  • Error: "API key not configured"
  • AI response never arrives

Possible Causes:
  1. API key not set (or set incorrectly)

  3. OpenRouter account issues

  ✓ Quota check: OpenRouter free tier has daily limits

ISSUE #6: LocalStorage Quota Exceeded
──────────────────────────────────────

Symptoms:
  • Error: "QuotaExceededError"
  • Forms suddenly fail to save
  • Navigation becomes slow

Possible Causes:
  1. Too much data stored (unlikely - app only uses ~1 MB)
  2. Browser storage quota full
  3. Other apps/tabs consuming storage
  4. Browser storage corrupted

Solutions:
  ✓ Clear old data: DevTools → Application → LocalStorage → Delete entries
  ✓ Export data: Copy important info before clearing
  ✓ Clear cache: DevTools → Application → Storage → Clear site data
  ✓ Restart: Close all browser tabs → Restart browser
  ✓ Check quota: JavaScript in console:
    navigator.storage.estimate().then(estimate => {
      console.log('Used:', estimate.usage, 'Quota:', estimate.quota);
    });

  ✓ Browser: Use Chrome, Firefox (Edge, Safari have issues)

Possible Causes:
  1. manifest.json missing or incorrect
  2. sw.js not properly registered
  3. HTTPS not configured (required for PWA)
  4. Browser requirements not met

Solutions:
  ✓ Production only: PWA install requires HTTPS
  ✓ Localhost: Works on http://localhost:*
  ✓ Verify: manifest.json exists & valid JSON
  ✓ Check: sw.js registration in index.html
  ✓ Browser: Chrome/Edge work best for PWA

8. TIPS FOR BEST RESULTS
================================================================================

A. PERFORMANCE OPTIMIZATION
─────────────────────────────

1. Browser Cache Management
   • Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
   • Clear cache monthly for faster updates
   • Browser handles caching automatically after first load

2. LocalStorage Optimization
   • Export data regularly (backup)
   • Clear completed transactions quarterly
   • Archive old crop data
   • Reduces storage size, improves speed

3. Multi-Tab Usage
   • Close unnecessary browser tabs
   • Frees up RAM → smoother operation
   • Recommended: Keep max 3 tabs open while using app

4. Device Optimization
   • Close background apps
   • Close browser notifications
   • Disable browser extensions (except Live Server)
   • Restart device weekly

5. Image Optimization
   • For quality grading: Use clear crop photos
   • Recommended: 2-5 MB JPEG images
   • Avoid: 20+ MB raw images (slow upload)
   • Format: JPG, PNG (not TIFF, BMP)

B. DATA MANAGEMENT BEST PRACTICES
──────────────────────────────────

1. Regular Backups
   • Export LocalStorage monthly
   • Manual: DevTools → Application → LocalStorage → Export
   • Store: Google Drive, email, USB
   • Recover: Copy-paste if browser crashes

2. Data Organization
   • Use descriptive crop names
   • Record activity dates consistently
   • Add notes/observations
   • Helps: AI recommendations more accurate

3. Photo Documentation
   • Crop growth: Photo weekly
   • Disease symptoms: Clear close-up
   • Field condition: Wide-angle shot
   • Timestamp: Helps track progress

4. Transaction Recording
   • Note all market prices
   • Record buyer details
   • Keep shipping proofs
   • Useful: Price forecasting accuracy

5. Seasonal Planning
   • Plan at season start
   • Review planning quarterly
   • Adjust based on actual performance
   • Track: Profit improvements year-on-year

C. AI FEATURE OPTIMIZATION
────────────────────────────

1. Quality Grading
   • Best: Well-lit, clear crop photos
   • Angle: Flat surface, natural lighting
   • Include: Size reference (coin, hand)
   • Avoid: Blurry, dark, close-up-only photos

2. Disease Detection
   • Describe: Specific symptoms
   • Location: Which part of plant?
   • Duration: How long showing symptoms?
   • Example: "Yellow spots on bottom leaves for 3 days"
   • Better AI predictions = Better treatment

3. Crop Recommendations
   • Provide: Exact NPK values (soil test report)
   • Include: Land size and water availability
   • Specify: Local climate/weather patterns
   • Mention: Budget for fertilizers/seeds
   • More details = Better crop selection

4. Weather Integration
   • Check: Daily weather on Weather page
   • Plan: Weekly activities around forecast
   • Track: Rainfall for irrigation planning
   • Alert: Monitor severe weather warnings

5. Price Forecasting
   • Historical data: Review past trends
   • Market news: Check international prices
   • Regional: Compare with neighboring states
   • Timing: Sell when prices peak (use alerts)
