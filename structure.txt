================================================================================
                    AGRI-OS (KrishiOS) PROJECT STRUCTURE
================================================================================

PROJECT DIRECTORY TREE:
======================

agri-os/
├── index.html              (Main HTML file - Layout, Navigation & UI)
├── style.css               (Stylesheet - Dark green farm theme with responsive design)
├── data.js                 (Static Database - Crop info, market prices, crop library)
├── app.js                  (Main Application Logic - 16 pages + AI integration)
├── sw.js                   (Service Worker - PWA offline support)
├── manifest.json           (PWA Manifest - App metadata & configuration)
├── README.md               (Project Documentation)
└── .git/                   (Version control repository)

================================================================================
                            FILE DESCRIPTIONS
================================================================================

1. INDEX.HTML (12 KB)
   ─────────────────
   • Main entry point of the application
   • Contains HTML structure for all 16 pages
   • Uses single-page application (SPA) approach with hidden divs
   • Includes navigation header, sidebar, and main content area
   • Responsive layout for mobile, tablet, and desktop
   • Bootstrap form elements and interactive components

2. STYLE.CSS (24 KB)
   ──────────────────
   • Complete styling for entire application
   • Dark green agricultural theme (#0d1f16 background, #2d6a4f accents)
   • Responsive CSS with media queries for mobile optimization
   • Component styling: cards, buttons, modals, tables, charts
   • Animations and transitions for better UX
   • Form styling and input validation visual feedback

3. DATA.JS (16 KB)
   ────────────────
   • Crop database with 12 major Indian crops
   • Market prices and MSP (Minimum Support Price) data
   • Weather information and historical data
   • Government schemes database
   • Sample soil analysis data
   • Equipment rental catalog
   • Storage facility information

4. APP.JS (200 KB)
   ────────────────
   • Core application logic and functionality
   • Handles all 16 page navigation and interactions
   • OpenRouter API integration for AI features
   • Features:
     ✓ User registration (Farmer/Buyer)
     ✓ Crop planning and tracking
     ✓ Soil analysis with AI recommendations
     ✓ Disease & pest detection
     ✓ Quality grading with image upload
     ✓ Marketplace and auction management
     ✓ Weather intelligence
     ✓ Financial planning
     ✓ Community chat with voice assistant
     ✓ Admin dashboard
   • LocalStorage management for data persistence
   • Real-time price updates and calculations

5. SW.JS (4 KB)
   ─────────────
   • Service Worker for Progressive Web App (PWA)
   • Enables offline functionality
   • Caches static assets
   • Handles background sync

6. MANIFEST.JSON (1.5 KB)
   ───────────────────────
   • PWA manifest file
   • App metadata: name, description, icons
   • Theme colors and display settings
   • App categories: agriculture, productivity, finance
   • Language: English (India)

================================================================================
                          KEY TECHNOLOGIES USED
================================================================================

Frontend:
  • HTML5 (Semantic markup)
  • CSS3 (Flexbox, Grid, Animations)
  • Vanilla JavaScript (ES6+)
  • No external frameworks or libraries

APIs & Services:
  • OpenRouter API (Free AI models - Llama 3.1, Mistral, Gemma)
  • LocalStorage API (Data persistence)
  • Service Workers (Offline support)

Data Storage:
  • Browser LocalStorage (User data, preferences, transaction history)
  • Session Storage (Temporary session data)

================================================================================
                            PAGE STRUCTURE
================================================================================

16 Main Pages (Accessible via Navigation):
  1. Dashboard - Overview & price alerts
  2. Registration - Farmer/Buyer registration
  3. Crop Planning - Plan crops with AI yield prediction
  4. Crop Library - Browse 12 Indian crops with details
  5. Soil Analysis - Analyze soil NPK with AI recommendations
  6. Growth Tracking - Track crop growth & satellite monitoring
  7. Disease & Pest - Detection with treatment recommendations
  8. AI Quality Grader - Upload crop photo for A/B/C grading
  9. Marketplace - Direct crop selling to buyers
  10. Auction - Bid-based crop selling
  11. Weather - Weather forecast & water management
  12. Market Intelligence - Price forecasting & demand analysis
  13. Storage & Logistics - Warehouse discovery & transport
  14. Finance & Schemes - Government schemes, insurance, equipment rental
  15. Community - Chat, expert consultation, knowledge hub
  16. Admin Dashboard - Supply chain tracking & analytics

================================================================================
                         DATA FLOW ARCHITECTURE
================================================================================

User Input
    ↓
App.js (JavaScript Logic)
    ↓
LocalStorage (Data Persistence)
    ↓
UI Update (DOM Manipulation)
    ↓
[Optional] OpenRouter API Call for AI Features
    ↓
Response Processing & Display

User Registration Data → Stored in LocalStorage
User Actions → Processed by app.js
AI Requests → Sent to OpenRouter (if API key configured)
Responses → Stored & Displayed in UI

================================================================================
                          DEPLOYMENT STRUCTURE
================================================================================

For Web Server Deployment:
  agri-os/
    └── All 6 files placed in root directory of web server
    └── Access via: http://domain-name/index.html
    └── Ensure HTTPS for API security

For Local Development:
  1. Use VS Code Live Server extension
  2. Or use Python's http.server
  3. Or use Node.js http-server

For Mobile App (Using Cordova/Electron):
  1. Wrap the project with Cordova CLI
  2. Build APK or iOS app
  3. Customize manifest.json for app store
