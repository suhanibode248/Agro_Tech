================================================================================
                    AGRI-OS (KrishiOS) PROJECT STRUCTURE
================================================================================

PROJECT DIRECTORY TREE:
======================

agri-os/
├── index.html              (Main HTML file - Layout, Navigation & UI)
├── style.css               (Stylesheet - Dark green farm theme with responsive design)
├── data.js                 (Static Database - Crop info, market prices, crop library)
├── app.js                  (Main Application Logic - 16 pages + AI integration)
├── sw.js                   (Service Worker - PWA offline support)
├── manifest.json           (PWA Manifest - App metadata & configuration)
├── README.md               (Project Documentation)
└── .git/                   (Version control repository)


 

================================================================================
                         DATA FLOW ARCHITECTURE
================================================================================

User Input
    ↓
App.js (JavaScript Logic)
    ↓
LocalStorage (Data Persistence)
    ↓
UI Update (DOM Manipulation)
    ↓
[Optional] OpenRouter API Call for AI Features
    ↓
Response Processing & Display

User Registration Data → Stored in LocalStorage
User Actions → Processed by app.js
AI Requests → Sent to OpenRouter (if API key configured)
Responses → Stored & Displayed in UI


Performance: Lightweight (~258 KB total)
PWA: Service Worker + Offline support

================================================================================
                         END OF STRUCTURE
================================================================================
